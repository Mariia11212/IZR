## Опис проєкту

Проєкт складається з двох основних логічних частин:

1.  **Власні структури даних:** Динамічний масив (`MyDynamicArrayBasedList<T>`), що слугує гнучкою альтернативою стандартним колекціям .NET.
2.  **Геометрична бібліотека:** Набір класів для представлення 2D-фігур, таких як `Point`, `Vector`, `Rectangle`, `Circle` та виконання над ними трансформацій.

## Документація 
Для забезпечення високої якості коду, його подальшої підтримки та простоти використання було приділено значну увагу документуванню. Весь публічний API (класи, методи, властивості) ретельно задокументовано з використанням стандартного для платформи .NET підходу.

### Підхід та інструментарій

Основою для документування коду слугують **XML-коментарі (`///`)**, вбудовані безпосередньо в C#. Цей підхід забезпечує низку ключових переваг:

*   **Інтеграція з IDE**: Visual Studio та інші середовища розробки використовують ці коментарі для відображення спливаючих підказок **IntelliSense**. Це дозволяє розробникам миттєво отримувати інформацію про методи, їхні параметри та очікувані результати, не покидаючи редактор коду.
*   **"Жива" документація**: Оскільки документація знаходиться поруч із кодом, її значно легше підтримувати в актуальному стані при внесенні змін.
*   **Автоматична генерація**: На основі XML-коментарів можна автоматично створювати професійні зовнішні файли документації (наприклад, у форматі HTML) за допомогою таких інструментів, як **DocFX**.

### Структура та повнота покриття

Документацією покрито всі ключові аспекти публічного API, що визначають контракт взаємодії з бібліотекою:

*   **Призначення (`<summary>`)**: Кожен публічний клас, структура, метод та властивість мають чіткий опис свого призначення.
*   **Параметри (`<param>`)**: Для кожного параметра методу вказано його роль, очікувані значення та можливі обмеження.
*   **Значення, що повертаються (`<returns>`)**: Детально описано, що саме повертає метод, особливо в неочевидних випадках (наприклад, повернення `-1` методом `IndexOf`, якщо елемент не знайдено).
*   **Винятки (`<exception>`)**: Задокументовано всі типи винятків, які може генерувати метод, та умови, за яких вони виникають.


## Юніт-тестування

Для перевірки коректності роботи реалізованих модулів було створено комплексний набір юніт-тестів.

### Фреймворк

Тести розроблено з використанням фреймворку **NUnit**, який є одним зі стандартів для тестування у .NET-екосистемі.

### Тестове покриття

Юніт-тестами покрито наступні ключові модулі та функціональність:

*   **`MyDynamicArrayBasedList<T>`**: Створення, операції з елементами, пошук, автоматичне розширення та обробка граничних випадків.
*   **`Vector` та `Point`**: Створення, математичні обчислення та операції трансформації.
*   **`Translation`**: Застосування трансформації переміщення до геометричних фігур.
*   **`Rectangle` та `Circle`**: Валідація даних, обчислення властивостей та застосування трансформацій.

### Детальний опис тестового покриття

#### `MyDynamicArrayBasedList<T>` (Динамічний масив)

*   **Створення та ініціалізація:**
    *   Перевіряється, що щойно створений список є порожнім (`Count` дорівнює 0).
*   **Додавання та зміна розміру:**
    *   Тестується, що метод `Add` коректно збільшує лічильник елементів.
    *   Перевіряється автоматичне розширення внутрішньої ємності при перевищенні початкового ліміту.
*   **Доступ та пошук елементів:**
    *   Перевіряється коректність отримання елемента за валідним індексом (`GetAt`) та генерація винятку `ArgumentOutOfRangeException` для невалідних індексів.
    *   Тестується, що `IndexOf` правильно знаходить індекс першого входження елемента та повертає `-1`, якщо його немає.
    *   Перевіряється робота `FindFirst` для пошуку елемента за умовою.
*   **Вставка та видалення:**
    *   Перевіряється коректність вставки (`InsertAt`) та видалення (`RemoveAt`) елементів з різних позицій у списку.
*   **Очищення та ітерація:**
    *   Тестується, що метод `Clear` повністю очищує список.
    *   Перевіряється, що ітератор (`foreach`) коректно працює як на заповненому, так і на порожньому списку.

#### `Vector` та `Point` (Вектор та Точка)

*   **Ініціалізація:**
    *   Перевіряється, що конструктори правильно встановлюють координати.
*   **Математичні операції:**
    *   Тестується коректність обчислення довжини вектора та відстані між точками.
*   **Трансформації та незмінність (Immutability):**
    *   Перевіряється, що методи `Scale` та `Translate` повертають **новий** об'єкт, не змінюючи оригінальний.

#### `Rectangle` та `Circle` (Прямокутник та Коло)

*   **Валідація в конструкторі:**
    *   Тестується генерація винятку `ArgumentException` при спробі створити фігуру з невалідними параметрами (нульові або від'ємні розміри/радіус).
*   **Обчислення властивостей:**
    *   Перевіряється коректність розрахунку площі (`GetArea`) та периметра (`GetPerimeter`).
*   **Застосування трансформацій:**
    *   Тестується коректність зміщення фігур за допомогою `Translation`.
    *   Перевіряється обробка `null`, `NaN` та `Infinity` значень під час трансформацій.

## Вдосконалення коду тестів

Під час розробки тестів було застосовано низку сучасних практик для забезпечення їхньої якості та надійності:

*   **Усунення дублювання коду**: Повторювана ініціалізація об'єктів винесена у метод `[SetUp]`.
*   **Підвищення читабельності**: Назви тестів було спрощено, а зайві коментарі та невикористовувані простори імен видалено.
*   **Оптимізація тестів**: Схожі сценарії перевірки винятків об'єднано за допомогою атрибута `[TestCase]`.
*   **Розширення тестового покриття**: Додано тести для важливих граничних випадків (робота з початком/кінцем списку, ітерація по порожньому списку).
*   **Гарантія незмінності (Immutability)**: Додано перевірки, що методи `Scale` та `Translate` повертають **новий екземпляр** об'єкта.
*   **Надійність обробки даних**: Реалізовано тести, що перевіряють коректну обробку невалідних (`NaN`, `Infinity`) або `null` трансформацій.
*   **Стандартизація коду**: Точність порівняння чисел з плаваючою комою стандартизовано за допомогою константи (`Eps = 1e-3`).
