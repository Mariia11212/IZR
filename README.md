# Лабораторна робота: Юніт-тестування структур даних та геометричної бібліотеки

## Опис проєкту

Проєкт складається з двох основних логічних частин:

1.  **Власні структури даних:** Реалізація базових колекцій, таких як динамічний масив.
2.  **Геометрична бібліотека:** Набір класів для представлення 2D-фігур (точка, вектор, прямокутник, коло) та виконання над ними афінних перетворень.

## Юніт-тестування

Основна мета цієї лабораторної роботи — створення комплексного набору юніт-тестів для перевірки коректності реалізованих модулів.

### Фреймворк

Тести розроблено з використанням фреймворку **NUnit**, який є одним зі стандартів для тестування у .NET-екосистемі.

### Покриті модулі

На даний момент юніт-тестами покрито наступні ключові модулі проєкту:

*   **`MyDynamicArrayBasedList<T>`**:
    *   Створення, ініціалізація та очищення списку (`Clear`).
    *   Додавання, вставка та видалення елементів (`Add`, `InsertAt`, `RemoveAt`).
    *   Доступ до елементів та їх пошук (`GetAt`, `IndexOf`, `FindFirst`).
    *   Автоматичне розширення ємності масиву.
    *   Перевірка граничних випадків (невалідні індекси, робота з порожнім списком).

*   **`Vector`**:
    *   Коректність створення та ініціалізації.
    *   Обчислення довжини (магнитуди) та операція масштабування (`Scale`).

*   **`Point`**:
    *   Коректність створення та ініціалізації.
    *   Обчислення відстані до іншої точки (`GetDistanceTo`).
    *   Операція переміщення за допомогою вектора (`Translate`).

*   **`Rectangle` та `Circle`**:
    *   Валідація вхідних даних у конструкторі (ширина, висота та радіус мають бути додатними).
    *   Обчислення площі (`GetArea`) та периметра (`GetPerimeter`).
    *   Застосування трансформацій.

## Вдосконалення коду тестів

Під час розробки тестів було застосовано низку сучасних практик для забезпечення їхньої якості, надійності та читабельності:

*   **Усунення дублювання коду**: Повторювана ініціалізація об'єктів винесена у метод `[SetUp]` для кожного тестового класу, що зробило тести чистішими та лаконічнішими.
*   **Підвищення читабельності**: Назви тестів було спрощено для кращого розуміння, а зайві коментарі (`// Тест #N`) та невикористовувані простори імен (`using`) — видалено.
*   **Оптимізація тестів**: Схожі сценарії перевірки винятків (наприклад, для невалідних індексів або розмірів) об'єднано за допомогою атрибута `[TestCase]`, що зменшило обсяг коду.
*   **Розширення тестового покриття**: Додано тести для важливих граничних випадків:
    *   Вставка та видалення елементів на початку та в кінці списку.
    *   Коректна поведінка ітератора (`foreach`) на порожньому списку після виклику методу `Clear()`.
    *   Перевірка, що `FindFirst` повертає саме *перший* із кількох елементів, що відповідають умові.
*   **Гарантія незмінності (Immutability)**: Для методів `Scale` та `Translate` додано перевірку, яка підтверджує, що вони повертають **новий екземпляр** об'єкта, а не змінюють існуючий.
*   **Надійність обробки даних**: Для `Rectangle` та `Circle` додано тести, що перевіряють коректну обробку невалідних (`NaN`, `Infinity`) або `null` трансформацій.
*   **Стандартизація коду**: Точність порівняння чисел з плаваючою комою стандартизовано за допомогою константи (`Eps = 1e-3`) для послідовності та ясності.
